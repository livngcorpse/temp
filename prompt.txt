PROMPT 1 — QUICKDROP CORE
Build a modern full-stack web application called “QuickDrop Core” — a temporary file sharing platform focused on speed, simplicity, realtime feedback, and clean UX.

The application should feel like a polished production-grade product similar to WeTransfer, but faster, lighter, and more developer-focused.

━━━━━━━━━━━━━━━━━━
CORE PURPOSE
━━━━━━━━━━━━━━━━━━

Users can:

upload files instantly
generate temporary share links
share files without requiring signup
set expiration times
optionally protect files with passwords
receive realtime updates about uploads and downloads

The platform should prioritize:

fast uploads
minimal UI
realtime feedback
smooth interactions
temporary storage
automatic cleanup

The application should remain lightweight, anonymous-first, and optimized for rapid file sharing.

3. REALTIME FEATURES
━━━━━━━━━━━━━━━━━━

Use Socket.IO throughout the application.

Realtime capabilities:

A. Live Upload Progress

progress bars update in realtime
upload percentages update smoothly
upload states synchronized properly

B. Download Notifications

Uploader receives live notification:

“Someone downloaded your file.”

C. Active Viewer Count

Display live viewer presence:

“3 people viewing this file.”

D. Live Expiration Countdown

Every file page should display a realtime countdown:

“Expires in 01:23:42”

Realtime systems should remain isolated from core upload logic so the application still functions if sockets fail.
